Base instructions, generated context, user turn

Initial context is a generated prefix, not a pile of old reminders.

Codex separates immutable model instructions from model-visible history items. AGENTS.md is loaded into session user instructions, rendered as a contextual user block, and re-injected when the runtime needs a fresh context baseline.

01
Base instructions are outside history.
They travel as the Responses API instruction field, resolved from config, resume metadata, or model defaults.
02
Initial context is history.
It is emitted as developer and user messages before the real user turn when the context baseline is missing.
03
Compaction regenerates context.
Old prefix entries are dropped; the current session context is injected again on the next suitable boundary.

AGENTS.md Assembly Chain

AGENTS.md content is preassembled into SessionConfiguration.user_instructions. Project docs are discovered along the cwd path and concatenated root-to-leaf.

Per directory, Codex prefers AGENTS.override.md, then AGENTS.md, then configured fallback names. The first matching regular file in each directory wins.

Initial Context Buckets

build_initial_context() builds three visible buckets: one aggregated developer item, optional separate developer items, and one aggregated contextual user item.

The developer and user buckets are multi-content messages. Each section becomes its own content item inside the same role message.

What Compaction Does To These Buckets

Prefix context is treated as generated inventory. Compaction drops old generated blocks and installs compacted history; fresh context is then injected at the next controlled boundary.

This is why AGENTS.md appears to survive: the old contextual user message is filtered out, but the session’s loaded AGENTS/user-instruction string is rendered again.
local

Local compaction

Collects real user messages and the summary. AGENTS and environment prefix entries are filtered as non-real user content.

remote

Remote compaction

Retains only installed retention shape. Raw system and developer messages are not installed as retained history.

AGENTS

Reread caveat

Compaction reuses the session-loaded AGENTS content. It is not evidence that edited AGENTS files were reread from disk mid-thread.
