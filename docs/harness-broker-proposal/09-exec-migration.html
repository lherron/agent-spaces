Relationship to hrc-server launch/exec.ts

What to extract, move, discard, and refactor from the existing one-shot wrapper.

Verdict

exec.ts is suitable as the basis for the one-shot broker runner, but only after cutting out HRC-specific state and callback assumptions.

It has the right mechanics: launch file, env scrub, spawn, stdio selection, signal forwarding, Codex output/app-server handling, callback/spool, degraded completion, and exit callback. The extraction should preserve the mechanics and remove HRC identity semantics.

Extraction sequence

  1. Create protocol package with HarnessInvocationSpec, event types, and a compatibility parser for current HrcLaunchArtifact.
  2. Copy exec.ts into broker package and change its input to normalized envelope/spec.
  3. Replace HRC callback endpoints in broker with an EventSink interface.
  4. Move Codex JSONL and app-server event mapping into broker drivers; normalize events.
  5. Update HRC to write HrcBrokerLaunchEnvelope and spawn harness-broker run instead of hrc-server/src/launch/exec.ts.
  6. Delete direct spaces-harness-codex/codex-session import from HRC.
  7. After SDK broker driver lands, delete HRC sdk-adapter.ts runtime execution path.
