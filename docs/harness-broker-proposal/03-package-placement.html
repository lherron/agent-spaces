Package Placement

Where harness-broker should live, what it should depend on, and what should depend on it.

Recommendation

Place broker inside Agent Spaces now: agent-spaces/packages/harness-broker. Add agent-spaces/packages/harness-protocol as a pure protocol package.

This gives the broker direct access to existing harness driver code without importing the umbrella agent-spaces client. It also makes the ownership signal clear: Agent Spaces owns harness normalization; HRC owns orchestration and durability.

Extraction path to standalone later

Keep the public package names and schemas extraction-friendly. The later standalone package can be created by moving harness-protocol, harness-broker, and driver packages into a new workspace. Avoid referencing Agent Spaces internal path aliases in broker code. The protocol should remain vendor-neutral enough that HRC and future consumers do not need to know Agent Spaces exists at runtime.
