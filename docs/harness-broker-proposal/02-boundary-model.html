Boundary Model

Exact ownership of compile/materialization, broker invocation, and HRC runtime control.

Boundary rule

Compile before broker invocation. The broker consumes an already-compiled harness invocation and owns only invocation mechanics.

This should be enforced in package dependencies, schema naming, and protocol commands. The broker has no command that accepts a placement, target name, space spec, project root, or harness preference to resolve. It only accepts a concrete invocation spec.

ASP compiler/materializer

Resolve placement and dry-run placement context.
Choose provider, harness ID, frontend, model, interaction/io mode.
Detect adapter, validate compatibility, materialize spaces, compose bundle.
Prepare Codex home, SDK bundle refs, system/priming prompts, brain/tool env.
Build argv/env/cwd/driver metadata and declared capabilities.
Must not: spawn child processes, persist HRC runs, assign runtime IDs, hold in-flight sessions for HRC.

Harness broker

Validate HarnessInvocationSpec and optional broker launch options.
Scrub inherited env and spawn/attach/drive process, pty, pipes, RPC/app-server, or SDK session.
Accept input/steer/append/interrupt/stop requests and map to driver capability.
Normalize assistant/tool/input/continuation/exit/error events.
Return final continuation/result data; emit degraded outcomes when finality is ambiguous.
Must not: resolve placement, choose Codex vs Pi vs Claude, materialize bundles, mutate HRC DB, enforce generation fences.

HRC runtime controller

Own host sessions, runs, runtime records, launch records, durable IDs, generations.
Choose runtime transport/lifecycle policy: tmux, headless, SDK/resident, wait/detach.
Create broker launch envelope, callback/spool sink, and supervision policy.
Persist broker events, derive semantic turn events, update continuation on success policy.
Reject stale callbacks, replay spool, reconcile startup state, mark zombies/orphans.
Must not: contain provider/frontend maps, SDK driver logic, Codex app-server JSON-RPC, brain/tool env prep.
