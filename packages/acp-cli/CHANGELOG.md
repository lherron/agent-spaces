# acp-cli — Changelog

## 0.1.0 — 2026-04-27

Migrated from a bespoke hand-rolled argv parser (`cli-args.ts`) to
**Commander v14** as part of the Commander CLI Upgrade project.

### Changed

- **Parser:** replaced `cli-args.ts` (168 LOC) with Commander v14 command
  tree. All verbs, flags, and positional arguments are unchanged.
- **Help output:** `acp --help` and per-command help is now auto-generated by
  Commander instead of 60+ hand-written `renderXxxHelp` helpers. Formatting
  differs; content is equivalent.
- **Exit codes:** usage errors (bad flag, missing argument) now consistently
  exit **2** via `CliUsageError`; runtime/server errors exit **1**. This
  matches the pre-existing convention but is now enforced centrally.
- **Dependencies:** added `commander ^14.0.0` and workspace `cli-kit`.

### Removed

- `src/cli-args.ts` — bespoke parser deleted.
- ~60 `renderXxxHelp` functions from `src/cli.ts`.

### Notes

- **No JSON output shape changes.** All structured output is bit-identical.
- `src/cli-runtime.ts` retained — it contains acp-specific error handling
  (`AcpClientHttpError`, `AcpClientTransportError`, `CliServerError`,
  `writeCommandOutput`) that is not duplicated in cli-kit.
